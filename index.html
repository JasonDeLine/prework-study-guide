HTML Notes

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The header element must be used numerically with h1 being the first and largest.

CSS Notes

  • Inline CSS – defines a style in the same line of code as what it is styling
  • Internal Style Sheet – contains CSS rules for the webpage in the head section of the HTML file which defines rules that only apply to that page. Can configure/define classes and IDs which identify elements that style multiple elements in the page code
  • External Style Sheet – Most common. Maintains CSS rules in a separate file from the HTML file, and makes design changes easier

Command Line Notes

  • CD: change directory. Can be followed by the directory name to which you wish to navigate
  • CD ..: changes directory up to the parent directory
  • LS: lists all files with the current directory
  • Git add -A: This command stages ALL (-A) changes made to a branch prior to pushing those changes to git
  • Git commit -m “commit message here”: This commend allows a user to add a commit message to the changes they are pushing
  • Git pull origin main: this makes sure that your local main branch and the git main branch are the same and up to date prior to pushing new changes
  • Git push origin [name of working branch]: Allows users to push changes onto the working branch they created where [name of working branch] is replaced with the name of the banch to which we are pushing the change/s
  • Git status: shows the status of the current branch on which you are working
